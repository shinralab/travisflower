Module 0: The three core ideas

1) Values

A value is data: number, string, boolean, object, etc.

2) Variables

A variable is a label pointing to a value.

3) Control flow

If this, do that. Repeat this. Stop when done.

Old style. Avoid unless maintaining old code.

Module 3: Operators you actually use

Math

Comparison

  • === equal (use this)
  • !== not equal
  • > < >= <=
Avoid == unless you know coercion rules.

Logical

&& and, || or, ! not

Assignment shortcuts

x += 1; // same as x = x + 1

Module 6: Functions

A function is a reusable mini-program.

Function declaration

  return a + b;

Arrow function

Return

Return gives the function’s output. Without return, the result is undefined.

Module 7: Arrays

Access

arr[0] // first item

Add / remove

  • push add end
  • pop remove end
  • shift remove start
  • unshift add start

The big 5 array methods

map (transform)

const doubled = nums.map(n => n * 2);

filter (keep matches)

const cheap = prices.filter(p => p < 50);

find (first match)

const user = users.find(u => u.id === 7);

some / every (any / all)

prices.some(p => p < 10);
prices.every(p => p > 0);

reduce (combine)

const sum = nums.reduce((acc, n) => acc + n, 0);
Exercises
  • Use map to turn ["a","b"] into ["A","B"].
  • Use filter to keep only even numbers.
  • Use reduce to total a cart: [9.99, 4.50, 2.00].

Module 10: Truthy / falsy

Falsy values: false, 0, "", null, undefined, NaN. Everything else is truthy.

Defaulting values

const name = input || "Guest";

Careful: || treats "" and 0 as “missing”. If you only want null/undefined to default, use ??.

const name = input ?? "Guest";
Exercises
  • What happens with 0 || 10?
  • What happens with 0 ?? 10?
  • When is ?? better than ||?

Module 14: Async basics

Some work takes time (network, files, timers). JS uses promises to represent “a value you’ll get later”.

async/await

async function load() {
  const res = await fetch("/data.json");
  const data = await res.json();
  return data;
}
Exercises
  • In plain English, what does await do?
  • What happens if fetch fails and you don’t catch errors?

Module 15: Best practices (80/20)

  • Use const by default
  • Use === not ==
  • Prefer small functions
  • Name variables clearly
  • Avoid deep nesting; use early return
  • Validate inputs at the edges

Early return example

function getDiscount(price) {
  if (price <= 0) return 0;
  if (price < 50) return 5;
  return 0;
}
